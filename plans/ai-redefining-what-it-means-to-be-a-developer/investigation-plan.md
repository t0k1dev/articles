# Investigation: AI Is Redefining What It Means to Be a Developer

## Purpose

Gather evidence, data points, and real-world examples to back
up the claims in the article. Every opinion should be grounded
in something concrete.

---

## Research areas

### 1. Historical comparisons

- How open source adoption changed team sizes and hiring
- Timeline: how long it took open source to go from niche to
  default (compare to AI tooling adoption speed)
- Cloud computing as a parallel shift — what jobs disappeared,
  what jobs appeared

**Sources to check:**
- [ ] GitHub Octoverse reports (open source growth data)
- [ ] Stack Overflow developer surveys (2020-2026 trends)
- [ ] "The Cathedral and the Bazaar" — revisit the core thesis

### 2. Productivity data

- Studies or reports measuring AI tool impact on developer
  output
- Time comparisons: how long specific tasks take with and
  without AI assistance
- Any data on solo developers shipping products faster

**Sources to check:**
- [ ] GitHub Copilot productivity research papers
- [ ] Google DeepMind developer productivity studies
- [ ] McKinsey / Accenture reports on AI in software teams
- [ ] Indie hacker communities — find real examples of solo
      launches that would have required a team before

### 3. Role changes and job market

- How job descriptions have changed (compare 2023 vs 2026
  postings for senior developer roles)
- New titles or responsibilities appearing in job listings
- Hiring trends: are teams getting smaller?
- What skills are companies actually screening for now?

**Sources to check:**
- [ ] LinkedIn job posting trends
- [ ] Hacker News "Who is hiring" threads — compare recent
      months to two years ago
- [ ] Engineering manager blog posts on team restructuring

### 4. The "80% is repetition" claim

- Find data on what developers actually spend time on
  (implementation vs design vs debugging vs meetings)
- Studies on code duplication and pattern reuse across
  projects
- Any surveys on developer satisfaction before/after AI
  tool adoption

**Sources to check:**
- [ ] Time-tracking studies on developer workflows
- [ ] JetBrains developer ecosystem surveys
- [ ] Microsoft Research papers on developer productivity

### 5. Codebase-as-context for AI

- How teams are adopting AGENTS.md, CLAUDE.md, Cursor rules
- Evidence that project structure affects AI output quality
- Examples of repos that document conventions for AI agents

**Sources to check:**
- [ ] OpenCode documentation on AGENTS.md
- [ ] Cursor docs on rules files
- [ ] Public repos with well-structured AI instruction files
- [ ] Blog posts from teams who restructured projects for AI

### 6. Skills gap and career impact

- Which developer skills are becoming more valuable
- Which skills are losing market value
- Salary trends for different specializations
- Bootcamp and education curriculum changes in response to AI

**Sources to check:**
- [ ] Stack Overflow salary data by skill
- [ ] Developer education platforms — what new courses launched
- [ ] Conference talk topics (compare 2024 vs 2026 lineups)
- [ ] Twitter/X threads from hiring managers on what they
      screen for now

### 7. Counterarguments and limits

This is critical for credibility. Investigate the cases where
the thesis does not hold:

- Large legacy codebases where AI tools struggle
- Regulated industries with strict code review requirements
- Security-sensitive systems where AI-generated code is a risk
- Teams that tried AI-first workflows and reverted

**Sources to check:**
- [ ] Post-mortems or blog posts about AI-generated bugs in
      production
- [ ] Security research on vulnerabilities in AI-generated code
- [ ] Enterprise engineering blogs on AI adoption challenges
